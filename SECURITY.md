# Security policy

## Supported versions

Security updates apply to the default branch (typically `main`) and, when relevant, the active development branch (e.g. `dev`). This repository is a **boilerplate**; forks and downstream projects should maintain their own versioning and patch strategy.

## Reporting a vulnerability

Please **do not** open a public issue for unfixed security problems.

- **GitHub**: use [Security advisories](https://docs.github.com/en/code-security/security-advisories) for this repository (private report to maintainers), if enabled for the repo.
- Otherwise, contact the repository owners through the channel indicated on the GitHub profile or organization page associated with this project.

Include enough detail to reproduce or assess the issue (affected component, steps, impact, suggested severity if known).

## Smart contracts and operational risk

Deploying contracts or running this stack involves **blockchain and key-management risks**. Follow operational best practices (key custody, network choice, audit scope appropriate to your deployment). This boilerplate does not replace a security audit for production use.
